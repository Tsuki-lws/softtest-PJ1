# 缺陷探索记录

本文件记录不纳入默认回归的探索性失败用例。默认 `mvn test` 仍只执行正式自动化回归用例；如需复现实验性失败结果，可在 `SoftwareTestingDemo/SoftwareTestingDemo` 目录下执行：

```bash
mvn "-Dtest=com.demo.defect.DefectExplorationCases" test
```

## 探索结果

手动运行结果：`Tests run: 6, Failures: 4, Errors: 2, Skipped: 0`。这些用例故意按“更严格/更理想的系统行为”断言，因此用于暴露风险，不计入正式回归通过率。

| 编号 | 失败场景 | 期望行为 | 当前表现 | 风险说明 |
|------|----------|----------|----------|----------|
| DEF-01 | 未登录访问后台用户管理页 `/user_manage` | 拒绝访问或要求管理员登录 | 可直接访问后台页面 | 后台管理入口缺少登录/权限校验 |
| DEF-02 | 不存在用户调用 `/checkPassword.do` | 返回 `false` | 触发空指针异常 | 参数不存在时缺少空值保护 |
| DEF-03 | 分页接口传入 `page=0` | 返回友好错误或自动修正页码 | `PageRequest.of(-1, ...)` 触发异常 | 分页边界值未做校验 |
| DEF-04 | 未登录调用 `/delOrder.do` 删除订单 | 拒绝删除 | 可直接删除指定订单 | 订单删除接口缺少登录/归属校验 |
| DEF-05 | 注册用户密码 | 不应明文存储 | 数据库保存明文密码 | 密码安全策略不足 |
| DEF-06 | 预约时长传入负数 | 拒绝提交 | 返回重定向，未按非法参数拦截 | 业务参数缺少合法性校验 |

## 汇报口径

可以在《测试总结》或汇报中表述为：

> 本轮正式自动化回归用例全部通过，说明已覆盖的核心路径符合预期。同时，探索性测试发现若干产品层面的改进点，包括后台权限校验、非法参数校验、密码安全策略和分页边界处理等。这些探索用例未纳入默认回归统计，作为后续优化方向记录。
