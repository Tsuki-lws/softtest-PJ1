# 缺陷记录

本文件记录当前测试中发现的问题用例。这 6 条用例已经纳入默认 `mvn test` 执行范围，可在 `SoftwareTestingDemo/SoftwareTestingDemo` 目录下直接复现：

```bash
mvn test
```

为避免与常规功能用例口径冲突，常规 controller 集成测试统一在合法用户或管理员会话下执行；本文件只记录“按预期应被拒绝，但当前实现未拒绝”或“应平稳处理，但当前直接报错”的问题场景。

这 6 条用例目前已分散并入以下集成测试类：

- `AdminUserControllerIntegrationTest`：DEF-01
- `PublicContentControllerIntegrationTest`：DEF-03
- `UserControllerIntegrationTest`：DEF-02、DEF-05
- `OrderControllerIntegrationTest`：DEF-04、DEF-06

## 执行结果

其中 6 条边界 / 鉴权 / 安全场景用例的执行结果为：`Tests run: 6, Failures: 4, Errors: 2, Skipped: 0`。这些结果表明当前系统在权限控制、边界值和安全性方面仍存在缺陷。

| 编号 | 失败场景 | 期望行为 | 当前表现 | 风险说明 |
|------|----------|----------|----------|----------|
| DEF-01 | 未登录访问后台用户管理页 `/user_manage` | 拒绝访问或要求管理员登录 | 可直接访问后台页面 | 后台管理入口缺少登录/权限校验 |
| DEF-02 | 不存在用户调用 `/checkPassword.do` | 返回 `false` | 触发空指针异常 | 参数不存在时缺少空值保护 |
| DEF-03 | 分页接口传入 `page=0` | 返回友好错误或自动修正页码 | `PageRequest.of(-1, ...)` 触发异常 | 分页边界值未做校验 |
| DEF-04 | 未登录调用 `/delOrder.do` 删除订单 | 拒绝删除 | 可直接删除指定订单 | 订单删除接口缺少登录/归属校验 |
| DEF-05 | 注册用户密码 | 不应明文存储 | 数据库保存明文密码 | 密码安全策略不足 |
| DEF-06 | 预约时长传入负数 | 拒绝提交 | 返回重定向，未按非法参数拦截 | 业务参数缺少合法性校验 |
